# 实践项目：学生成绩管理系统

> 本项目与课堂"奶茶门店系统"难度相当，但业务场景完全不同。通过独立完成此项目，巩固面向对象、文件操作、线程安全等核心知识点。

---

## 一、项目背景

你是一所中学的教务系统开发负责人。学校每学期末需要处理以下业务：

- 管理不同学科的考试成绩（语文、数学、英语）
- 每门学科有自己的满分标准和评分规则
- 不同学科有各自的成绩等级划分标准
- 多名老师同时录入成绩时不能数据冲突
- 每次成绩录入需要持久化保存
- 支持批量统计和优秀率计算

---

## 二、功能需求

### 2.1 核心功能

| 编号  | 功能      | 说明                             |
| --- | ------- | ------------------------------ |
| F1  | 学科信息管理  | 每门学科有名称、满分值、学生答卷               |
| F2  | 成绩录入    | 录入分数后自动验证合法性，超出满分拒绝录入          |
| F3  | 等级评定    | 不同学科有各自的等级划分规则（见 2.2）          |
| F4  | 成绩持久化   | 每次录入记录写入 `exam_records.txt` 文件 |
| F5  | 多线程安全录入 | 多名老师并发录入时数据不冲突                 |
| F6  | 批量统计    | 传入多份答卷实例，输出总分和平均分（多态）          |
| F7  | 优秀筛选    | 使用列表推导式筛选达到优秀的学生名单             |
| F8  | 成绩单生成器  | 使用 yield 惰性生成学生成绩单             |

### 2.2 各学科等级规则

| 学科  | 类名            | 满分   | 等级规则                       |
| --- | ------------- | ---- | -------------------------- |
| 语文  | `ChineseExam` | 150分 | ≥135优秀，≥120良好，≥90及格，<90不及格 |
| 数学  | `MathExam`    | 150分 | ≥140优秀，≥120良好，≥90及格，<90不及格 |
| 英语  | `EnglishExam` | 100分 | ≥90优秀，≥75良好，≥60及格，<60不及格   |

### 2.3 额外要求

- 语文有独有属性 `essay_score`（作文分，满分60）
- 数学有私有属性 `__bonus_points`（附加分，默认0），需提供 get_bonus_points/set_bonus_points
- 英语需重写 `print_report_card()` 方法，打印"听力/阅读/写作分项成绩"标语
- 所有学科新增 `get_grade(score)` 方法，根据分数返回等级（优秀/良好/及格/不及格）

## 四、具体实现任务

### 任务1：工具模块（grade_utils.py）

实现以下函数：

```
1. check_valid_score(score, max_score)  → 校验成绩是否在合法范围（0~满分）
2. calc_percentage(score, max_score)    → 计算得分率 = 分数/满分 × 100%
3. save_record(record_info)             → 使用 with 追加写入 exam_records.txt
4. read_all_records()                   → 使用 with 读取全部成绩记录
5. get_excellent_students(score_dict, threshold)  → 列表推导式筛选达到优秀的学生
6. report_card_generator(student_list)  → 生成器，yield 格式化成绩单字符串
7. input_score_thread_safe(student_name, subject, score)  → 线程锁安全录入成绩
8. multi_thread_input_test()            → 创建2个线程并发录入测试
```

**全局变量要求**：

- `student_records = {}`  # 全局共享成绩字典，格式：{"张三": {"语文": 0, "数学": 0}}
- `record_lock = threading.Lock()`

### 任务2：考试抽象基类（subjects/base_exam.py）

```python
类名: BaseExam (ABC)

类属性:
  - passing_rate = 0.6  # 及格率（60%）

实例属性:
  - subject_name: str   # 学科名称
  - max_score: float    # 满分值
  - student_name: str   # 学生姓名
  - __score: float      # 私有成绩，默认0

方法:
  - __init__(subject_name, max_score, student_name)
  - get_score() → float
  - input_score(score)                    # 录入成绩，超出满分抛异常
  - set_passing_rate(cls, rate)           # 类方法
  - check_student_name(name) → bool       # 静态方法
  - get_grade(score) → str                # 抽象方法（子类必须实现等级规则）
  - calc_weighted_score(weight) → float   # 计算加权分（如期末占70%）
  - print_report_card()                   # 通用成绩单打印
```

## 五、进阶挑战（选做）

### 挑战1：班级总分排名

新增函数 `calculate_class_ranking(records_dict)`，接收全班成绩字典，返回按总分降序排列的学生名单。

### 挑战2：成绩进步追踪

在 `BaseExam` 中新增实例方法 `compare_with_previous(previous_score)`，对比上次成绩，输出：

- 进步/退步分数
- 进步/退步百分比
- 等级是否变化

### 挑战3：学科均衡度分析

新增函数 `check_balance(student_scores)`，分析学生各科成绩的标准差，判断是否偏科：

- 标准差 < 10：各科均衡
- 10 ≤ 标准差 < 20：轻微偏科
- 标准差 ≥ 20：严重偏科，输出具体科目

### 挑战4：异常处理增强

在 `main.py` 中用 `try...except` 包裹所有可能抛异常的操作：

- 成绩超出满分时捕获 `ValueError`
- 参数类型错误时捕获 `TypeError`
- 给出友好的错误提示而非程序崩溃

## 九、参考提示

1. **先写工具函数，再写类，最后整合**——这是最稳妥的开发顺序
2. **边写边测试**——写完一个函数就用 `print` 验证，不要全部写完再运行
3. **参考课堂项目但不要被限制**——成绩系统的业务逻辑与奶茶系统不同，需要独立思考
4. **遇到报错不要慌**——看错误信息的最后一行，定位到具体文件和行号
5. 自己搭建框架，厘清项目逻辑，具体实现代码可以通过ai，输入你的提示词帮助优化，**禁止全部使用ai！！！！**
